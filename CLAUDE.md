# Project Rules

## Goal
사용자 요청을 적은 토큰으로 정확히 수행한다.

## Priorities
1. 토큰 최적화
2. 중복 작업 방지
3. 변경분 중심 검토
4. 실질 개선만 수행

## Working Rules
- 긴 배경 설명을 반복하지 말 것
- 현재 활성 작업만 기준으로 판단할 것
- 새 요청은 기존 작업과 먼저 비교할 것
- 같은 목표면 새 작업을 만들지 말고 병합할 것
- 변경 없는 영역은 다시 검토하지 말 것
- cosmetic 수준 개선은 기본적으로 생략할 것

## Task Limit
- main task 1개
- subtask 최대 1개

## Output
항상 아래 4개 섹션만 출력:
1. STATUS_SUMMARY
2. TASK_STATE_JSON
3. NEXT_ACTIONS
4. BOT_EVENTS

## Project Context
- 프로젝트명: PTW (Permit to Work / Plan to Work)
- 목적: PTW 시스템 기획 및 프론트엔드 프로토타입 제작
- 주요 작업:
  - Permit to Work: 작업 허가서 발급/승인/관리 시스템 기획
  - Plan to Work: 작업 계획 수립/공유/추적 시스템 기획
  - 프론트엔드 프로토타입 (HTML/CSS/JS 또는 React)
