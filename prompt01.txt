A. IA 원칙

  1) 대분류 (1개 선택)

- “무슨 작업인지” = 상세분류
- “무슨 장비를 쓰는지” = 장비
- “어떤 조건인지” = 작업속성 / 위험속성

장비는 상세분류 내부에 종속시키지 말고,

[화면/UX 요구사항]
==================================================

현재 프로토타입 흐름을 분석해서 가장 자연스러운 방식으로 반영하세요.
가능하면 아래 UX를 목표로 하세요.

1. 작업계획서 생성/편집 화면
- 상단: 페이지 제목, 현재 단계 또는 현재 선택 요약
- 좌측 또는 상단:
  - 대분류 선택
- 중앙:
  - 상세분류 선택
- 우측 또는 하단:
  - 장비 멀티선택
  - 작업속성 태그
  - 위험속성 태그

2. 상세분류 선택 UX
- 대분류 선택 시 해당 상세분류만 보이게
- 상세분류는 카드형 또는 리스트형 선택 UI
- 현재 선택된 상세분류가 명확하게 강조되어야 함

3. 장비 선택 UX
- 독립 패널
- 체크박스 기반 다중선택
- 그룹별 접기/펼치기 가능하면 좋음
- 선택된 장비 칩 표시

4. 요약 UX
- 화면 상단 또는 저장 버튼 근처에 현재 선택 상태 요약
- 예:
  - 대분류: 가설·구조보조 작업
  - 상세분류: 가설비계 작업
  - 장비: 지게차, 고소작업대(시저)
  - 작업속성: 해체
  - 위험속성: 야간, 개구부

5. 편집 UX
- 기존 저장값이 있다면 초기값으로 로드
- 수정 시 기존 선택 상태가 유지되어야 함

  - 추천 위험속성: 중량물, 인접작업

  - 추천 장비: 굴착기(백호)
  - 추천 작업속성: 굴착, 운반
  - 추천 위험속성: 인접작업, 협소공간

  - 추천 위험속성: 밀폐공간

추천값은 사용자를 돕기 위한 제안이어야 하며, 강제 로직이면 안 됩니다.
